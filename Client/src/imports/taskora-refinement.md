Refine the existing Taskora productivity platform design.

Do NOT redesign the theme or layout. Maintain the current dark purple aesthetic, typography, spacing, and component structure.

The goal is to extend the product with advanced productivity tracking, analytics, and a pricing system.

---

1. REMOVE SETTINGS OPTION

---

In the top-right user dropdown menu remove the "Settings" option.

The dropdown should only contain:

Profile
Logout

Do not change the dropdown style.

---

2. ADD PRICING PAGE

---

Add a new page called **Pricing**.

Access to this page should appear when clicking the **user section in the bottom-left corner of the sidebar** (where the user avatar and plan name appear).

When this area is clicked, a small menu should appear containing:

Pricing
Profile
Logout

Selecting **Pricing** opens the pricing page.

---

3. PRICING PAGE STRUCTURE

---

Design a clean SaaS pricing page with two subscription tiers.

Tier 1 — Free Plan

Features:
Basic task management
Focus timer
Session tracking
Limited focus insights
Basic analytics

Tier 2 — Pro Plan

Features:
Advanced focus insights
Full productivity analytics
Deep work pattern detection
Unlimited session history
Advanced statistics dashboard
Future AI productivity insights

Display pricing cards centered on the page.

Example layout:

Free Plan
$0 / month

Pro Plan
$8–12 / month

Highlight the Pro plan card with a subtle glow or accent border.

---

4. PHASE 3 — FOCUS BEHAVIOR TRACKING

---

Add behavior tracking indicators across the interface.

Focus session results should show:

Session duration
Interruptions
Focus quality score

Example session summary card:

Focus Summary

Session Duration: 25m
Interruptions: 2
Focus Score: 84%

This summary should appear after completing a focus session.

---

5. ADD SESSION TIMELINE

---

Add a **Session Timeline section** inside the Focus Insights page.

This section should display work blocks for the day.

Example layout:

9:00 — Deep Work — Auth Service — 25m
9:30 — Break — 5m
9:35 — Deep Work — API Docs — 25m

Visualize this either as:

• vertical list of sessions
or
• timeline blocks

---

6. ADD DAILY PRODUCTIVITY SUMMARY

---

Add a card inside the Focus Insights page called **Daily Productivity Summary**.

Display metrics such as:

Total Focus Time Today
Number of Sessions
Interruptions
Average Focus Score

Example:

Total Focus: 3h 40m
Sessions: 7
Interruptions: 3
Focus Score: 86%

7. PHASE 4 — PRODUCTIVITY ANALYTICS

---

Enhance the Focus Insights page with deeper analytics.

Add sections for:

Weekly Productivity Summary

Weekly Summary

---

8. TASK TIME ESTIMATION ACCURACY

---

Inside the Tasks page show comparison between:

Estimated Time
Actual Focused Time

Example:

Task: Build authentication API

Estimated: 60m
Actual: 85m
Accuracy: 70%

Display accuracy visually using a small progress indicator or badge.

---

9. HEATMAP RULE

---

The Focus Activity Heatmap must remain ONLY inside the **Focus Insights page**.

Do NOT place the heatmap inside Focus Mode.

---

FINAL GOAL

---

The application should feel like a **Deep Work Productivity Platform**, not just a task manager.

Focus on:

minimal UI
behavior tracking
productivity analytics
clear deep work insights
